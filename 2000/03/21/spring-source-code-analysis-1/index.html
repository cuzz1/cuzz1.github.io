Spring源码分析

Spring 是我们工作中最常用的框架，Spring使我们开发变得简单，工作几年只停留在使用 Spring 层面上，很多特性以及实现原理还知道，这次通过学习 Spring 源码加深自己对它的理解，同样学习
优秀的框架可以学到很多设计理念。

前言

Spring 最重要的是 IoC(Inversion of Control) 容器，IOC 容器就是具有依赖注入功能的容器，IoC 容器负责实例化、定位、配置应用程序中的对象及建立这些对象间的依赖。应用程序无需直接在代码中new相关的对象，应用程序由 IoC 容器进行组装。

上面这张图体现了 IoC 容器是如何装配一个 Bean 的，刚开始我们的解析配置（xml配置、注解配置）得到 BeanDefinition（Bean的定义信息），接着通过反射实例化，然后进行初始化得到一个完整的Bean，最后使用完成进行销毁。
